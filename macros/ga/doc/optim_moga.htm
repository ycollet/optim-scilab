optim_moga - Multiobjective Genetic Algorithm

Calling Sequence

[pop_opt,fobj_pop_opt,pop_init,fobj_pop_init] = optim_moga(ga_f,pop_size,nb_generation,p_mut,p_cross,init_func,Log,crossover_func,mutation_func,codage_func,selection_strategy,nb_couples,pressure)

Parameters

  • ga_f : the function we want to optimize (the prototype is f(x))
  • pop_size : the size of the population (default: 100)
  • nb_generation : maximum number of iterations (default: 10)
  • p_mut : mutation probability (default 0.01)
  • p_cross : crossover mutation (default 0.7)
  • init_func : a function which will produce an initial population (see the function init_func_default as an example)
  • Log : %T or %F. If %T, some informations will be displayed during the run of the multiobjective genetic algorithm
  • crossover_func : a function which will perform a crossover between two individuals (see crossover_func_default as an example)
  • mutation_func : a function which will perform a mutation on one individual (see mutation_func_default as an example)
  • codage_func : a function which will perform a translation of a population of individuals into an intermediate representation (binary strings for example). See codage_identity as an example
  • selection_strategy : To select what kind of selection strategy is applied to the pool of individuals. Currently, only the 'elitist' strategy is implemented
  • nb_couples : the number of couples produced during one generation (default 100)
  • pressure : the minimum efficiency value for the worse individual (default: 0.05 - the efficiency interval is 0 to 1)
  • pop_opt : the population of individuals obtained at the last generation
  • fobj_pop_opt : the set of corresponding objective functions values obtained at the last generation
  • pop_init : the initial population (optional parameter)
  • fobj_pop_init : the set of corresponding objective function values obtained for the initial population (optional parameter)

Description

The multiobjective genetic algorithm. This genetic algorithm uses tournament selection when building couples.
