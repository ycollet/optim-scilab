optim_ga - A Genetic Algorithm optimization method

Calling Sequence

[pop_opt,fobj_pop_opt,pop_init,fobj_pop_init] = optim_ga(f,pop_size,nb_generation,p_mut,p_cross,init_func,Log,crossover_func,mutation_func,codage_func,selection_strategy,nb_couples,pressure)

Parameters

  • f : the function we want to optimize (the prototype is f(x))
  • pop_size : the size of the population of individuals
  • nb_generation : the number of generations we want to run the genetic algorithm
  • p_mut : the probability of mutation (a real value usually around 0.1)
  • p_cross : the probability of crossover (a real value usually around 0.7)
  • init_func : a function which will produce an initial population (see the function init_func_default as an example)
  • Log : %T or %F. If %T, some informations will be displayed during the run of the genetic algorithm
  • crossover_func : a function which will perform a crossover between two individuals (see crossover_func_default as an example)
  • mutation_func : a function which will perform a mutation on one individual (see mutation_func_default as an example)
  • codage_func : a function which will perform a translation of a population of individuals into an intermediate representation (binary strings for example). See codage_identity as an example
  • selection_strategy : To select what kind of selection strategy is applied to the pool of individuals. Currently, only the 'elitist' strategy is implemented
  • nb_couples : the number of couples produced during one generation
  • pressure : the minimum efficiency value for the worse individual (default: 0.05 - the efficiency interval is 0 to 1)
  • pop_opt : the population of individuals obtained at the last generation
  • fobj_pop_opt : the set of corresponding objective function values obtained at the last generation
  • pop_init : the initial population (optional parameter)
  • fobj_pop_init : the set of corresponding objective function values obtained for the initial population

Description

A Genetic Algorithm optimization method.
